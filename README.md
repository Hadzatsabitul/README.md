## Issues Encountered

### Challenge: Unfamiliarity with Cursor IDE and GitHub

As someone new to these tools, the biggest challenge was not the installation itself, but understanding **what these tools are for** and **how to use them properly**.

**Problem:**
- I had never used Cursor IDE or GitHub before, so I wasn't sure how they worked or how they connected to each other.
- Terms like "repository", "commit", "push", and "extensions" were unfamiliar at first.
- I was also unsure how to open a GitHub repository inside Cursor and how to push files from my local machine to GitHub.

**How I Solved It:**
1. **Searched for tutorials on YouTube** — I looked up videos such as:
   - *"How to use Cursor IDE for beginners"*
   - *"How to create a GitHub repository"*
   - *"How to push code to GitHub step by step"*
2. **Explored the tools on my own** — I opened Cursor and clicked through the interface to understand the layout, the terminal, and the Extensions panel.
3. **Took it step by step** — Instead of trying to understand everything at once, I followed the instructions one by one, completing each step before moving to the next.
4. **Used the Extensions panel** — After understanding the interface, I was able to find and install Claude Code and Codex by searching directly in the Extensions panel inside Cursor.

**Result:**
By combining YouTube tutorials with hands-on exploration, I was able to complete all the required steps successfully. This process also helped me understand the basics of version control with Git and how AI-powered tools like Claude Code and Codex can assist in development workflows.
